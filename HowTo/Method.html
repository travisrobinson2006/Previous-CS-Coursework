Section 4

Method Parameter With the Facebook Graph API

The method parameter for the function determines what a user will be doing with their request. There are three potential options: GET, POST, and DELETE. The GET method is the method used for retrieving information; it's also the default, so is the one used in prior examples. It allows us to read a persons birthday, or their list of friends, or check on the list of admins for a group.

The POST method is what allows us to make Facebook posts. These can be creating a new photo album, putting a photo in an existing album, posting a video, or making a post to a persons wall.

The DELETE method, like its name implies, lets us delete information from Facebook, such as wall posts or photos.

All these methods, of course, require certain permissions. For example, you can't use the API to post to a random persons wall, and you can't delete users from an app (unless it's your own app, in which case you would have permission).

Looking at a code example to make a post to a news feed: 


function print() {
    FB.login(function(){
        FB.api('/me/feed', 'post', {message: 'CS290 is great!'});
        }, {scope: 'publish_actions'});
}

This code will print to my wall the message "CS290 is really great!" (PS, it is)

Here we use all of the tools or parameters that we've discussed so far. We follow the path to the users feed, which in this case will be the path to my feed, 'me/feed', then we pass it the 'post' parameter so that it knows we're posting information instead of retrieving, and then we come to the 'params' parameter, which in this case is an object containing a message, 'CS290 is really great!'

Please note this not a comprehensive guide to using the Facebook Graph API. There are a TON of roots,nodes, and params that will all do different things depending on which method is being used. What this How To should do though is give a good foundation. It shows how to make calls using the different API parameters and gives examples of some of the things that can be done with them; it should really be used in conjuntion with the documentation for your own projects as an example of how to get started and how to build from there.
